# 新手总教程（Workflow Studio 版）

这份文档只讲现在这一个主流程。  
你不需要再区分“新手向导 / 手动拼接 / 抓包导入”。

## 1. 下载安装

1. 打开 Releases：`https://github.com/ranjing666/script-generator/releases`
2. 下载最新的 `FengToolbox-*-Setup-x64.exe`
3. 双击安装
4. 打开 `风的工具箱`

如果你不想安装，也可以下载：
- `FengToolbox-*-Portable-x64.exe`

## 2. 先理解现在的软件怎么用

打开后你会看到三个区域：

- 左边：`项目库`
- 中间：`Workflow Studio 主工作台`
- 右边：`诊断 / 预览 / 生成结果`

你真正要做的事情现在有四种起点：

1. `从官网 URL 开始`
2. `从抓包开始`
3. `从模板开始`
4. `空白流程`

如果你之前已经有别人给你的流程文件，还可以：

5. `导入流程文件`

## 3. 最推荐的新手路径：从官网 URL 开始

适合：
- 你只有项目官网或任务页
- 还不会抓包
- 想先得到一个可编辑草案

步骤：

1. 左上角输入官网 URL
2. 点 `从官网 URL 开始`
3. 软件会自动分析页面、匹配适配器、生成 workflow 草案
4. 第 2 段 `来源材料` 会显示：
   - 页面标题
   - 分析方式
   - 适配器
   - 识别信号
   - 人工确认理由
   - 风险提示
   - 如果有来源快照，还可以直接点开 HTML 快照和分析摘要
5. 第 4 段检查步骤卡片，必要时手改
6. 右边先看诊断
7. 确认后再 `生成项目`

## 4. 次推荐路径：从抓包开始

适合：
- 你已经在浏览器里点过网站
- 想把测试网任务转成脚本
- 不会自己写请求

步骤：

1. 点左上角 `从抓包开始`
2. 选择抓包文件  
   优先 `HAR`，也支持 `Postman/cURL`
3. 软件会自动创建一个新流程项目
4. 中间第 2 段 `来源材料` 会显示：
   - 抓包类型
   - 原始文件名
   - 识别到的请求数 / 任务组数
   - 导入可信度
   - 风险提示
5. 中间第 3 段确认：
   - 账号来源
   - 登录方式
6. 中间第 4 段检查步骤顺序，必要时上移/下移/删除/复制
7. 右边点 `刷新预览`
8. 右边确认没有阻塞项后，点 `生成项目`

## 5. 什么时候用模板开始

适合：
- 你还没有抓包
- 只是想先搭一个可运行骨架
- 你知道自己大概属于哪类任务

步骤：

1. 左边模板下拉里选一个模板
2. 点 `从模板开始`
3. 中间补：
   - 项目名
   - 账号来源
   - 登录配置
   - 步骤里的 URL / body / 合约地址 / ABI
4. 右边看诊断和预览
5. 点 `生成项目`

## 6. 什么时候用空白流程

适合：
- 你已经很清楚目标网站怎么做
- 你想完全自己搭步骤

步骤：

1. 点 `空白流程`
2. 在第 3 段先选账号来源和登录方式
3. 在第 4 段右上角选步骤类型并点 `新增步骤`
4. 一步一步把 URL、请求体、链上参数填进去
5. 看右边诊断，修完阻塞项再生成

## 7. 中间这五段分别该怎么理解

### 1) 项目基础信息

你主要填：
- 项目名
- 输出目录
- 并发账号数
- 是否用代理
- 是否循环执行

对新手最重要的建议：
- 并发先填 `1`
- 循环先关掉

### 2) 来源材料

这段主要是“你这个流程是从哪来的”。

如果是抓包导入，它会告诉你：
- 这份抓包是否靠谱
- 哪些地方识别完整
- 哪些地方仍然危险

如果是官网 URL 分析，它会告诉你：
- 当前命中了哪个适配器
- 自动识别置信度高不高
- 是否需要人工确认
- 是否检测到验证码、钱包连接、前端加密

### 3) 账号与登录

这里决定脚本怎么拿身份。

常见三种账号来源：

- `账号密码文件`
- `Token 文件`
- `钱包私钥文件`

常见登录方式：

- `不预置登录`
- `直接读取 token`
- `普通请求登录`
- `钱包签名登录`

### 4) 步骤卡片流

这是脚本真正会执行的顺序。

你可以：
- 启用 / 禁用
- 上移 / 下移
- 复制
- 删除
- 改每一步的基础字段

### 5) 诊断 / 预览 / 生成

右边最重要。

你生成前先看两件事：

1. 有没有 `阻塞项`
2. 预览 JSON 里是不是你想要的 `project.config.json`

## 8. 生成出来后怎么运行

软件右边点 `生成项目` 后，会得到一个新目录。

进去按这个顺序做：

1. 打开 `00-先看这里-零基础说明.md`
2. 双击 `0-双击-运行前检查.bat`
3. 双击 `1-双击-安装依赖.bat`
4. 填 `.env`
5. 填 `data/` 里的账号文件
6. 双击 `2-双击-启动脚本.bat`

如果你完全不想记步骤：

- 直接双击 `2-双击-启动脚本.bat`

它现在会自动：
- 补装依赖
- 自动检查 `.env`
- 检查账号文件
- 失败时把错误写到 `logs/last-error.txt`

现在还多了一条路径：

- 你也可以留在 `风的工具箱` 里，直接点右侧的 `运行 / 暂停 / 继续 / 停止`

注意：
- 应用内运行依赖你已经先生成过一次项目目录
- 它本质上是在应用里托管输出目录中的 `main.js`

## 9. 最常见的几个文件怎么填

### `.env`

这里放敏感变量，比如：
- `RPC_URL`
- `CAPTCHA_TOKEN`
- 其他 `{{env.XXX}}` 变量

### `runtime.config.json`

这里主要放：
- 浏览器会话优先级
- 验证码策略
- 人工兜底策略
- 当前项目更推荐在桌面端跑还是独立项目跑
- 哪些步骤类型需要重点人工确认

大部分新手第一版不用先改。

### `artifacts/source-material.json`

这里会记录：
- 这个流程最初从哪里来的
- 自动分析命中了什么适配器
- 风险提示和人工复核原因
- 导出时一起带走了哪些来源文件

### `data/privateKeys.txt`

每行一个私钥。

### `data/tokens.txt`

每行一个 token。

### `data/accounts.txt`

每行一组账号，默认像这样：

```text
email@example.com|your_password
```

### `data/requestRows.txt`

只有你用了“批量提交行数据”步骤才会出现。  
每行一组参数，通常用 `|` 分隔。

## 10. 看不懂报错时先看哪里

顺序固定：

1. 右边 `诊断`
2. 生成目录里的 `运行前检查报告.txt`
3. 生成目录里的 `logs/last-error.txt`

## 11. 新手三条安全线

1. 私钥和 token 不要发给别人
2. 先用小号测试
3. 并发先从 `1` 开始

## 11. 如果你想把流程给别人

右边点：

- `导出 .fengflow.json`

这样你导出的不是最终脚本目录，而是“流程本身”。  
别人拿到后可以直接在软件里 `导入流程文件`，继续改，再生成。
